# GEO Audit Tool for GymBeam blog

A command-line application for auditing WordPress articles from a GEO / SEO
content-quality perspective.

The tool analyzes blog posts against a fixed set of criteria (definitions,
structure, facts, sources, metadata, etc.) and generates CSV and/or HTML reports
with scores and recommendations.

---

## Requirements

- Python **3.10+**

Python dependencies:
- `requests`
- `beautifulsoup4`

Python development dependencies:
- `pytest`

---

## Setup

### 1. Clone the repository

```
git clone https://github.com/Murrko-001/geo_audit.git
cd geo_audit
```
---

### 2. (Optional but recommended) Create a virtual environment
```
python -m venv .venv
```
Activate it:

- **Linux / macOS**
  ```bash
  source .venv/bin/activate
  ```

- **Windows**
  ```bat
  .venv\Scripts\activate
  ```

---

### 3. Install dependencies

Install the runtime dependencies required to run the application:
```
pip install -r requirements.txt
```

---

## Running the application

The application is executed using a small helper script located at the project
root.

### Basic usage

```bash
python geo_audit.py
```

If no output flags are provided, **both CSV and HTML reports are generated by
default**.
The generated reports are saved to the ```geo_audit/output``` folder.

---

### Command-line options

| Option         | Description |
|----------------|------------|
| `-h`, `--help`     | Show help message and exit |
| `--wp`         | Load articles from the WordPress REST API (default is local JSON) |
| `--per-page N` | Number of articles to analyze (default: `10`) |
| `--csv`        | Export report to CSV |
| `--html`       | Export report to HTML |

---

### Examples

Analyze 10 articles from WordPress and export both reports:

```bash
python geo_audit.py --wp
```

Analyze 5 articles from WordPress and export HTML only:

```bash
python geo_audit.py --wp --per-page 5 --html
```

Use local cached articles and export CSV only:

```bash
python geo_audit.py --csv
```

## Running tests

Unit tests are implemented using `pytest`.

Before running the tests, install the development dependencies:

```bash
pip install -r requirements-dev.txt
```

Run all tests from the project root:
```
pytest
```
